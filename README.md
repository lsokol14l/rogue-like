## О проекте

**Rogue21** — пошаговая консольная игра с процедурно генерируемыми подземельями. Цель — пройти 21 уровень, сражаясь с монстрами, собирая предметы и снаряжение. Каждое прохождение уникально: уровни генерируются заново, сложность растёт с глубиной.

После каждой попытки (успешной или нет) результат фиксируется в **таблицу рекордов**, отсортированную по количеству собранных сокровищ. Прогресс можно сохранить и продолжить при следующем запуске.

## Архитектура

Приложение построено на **многослойной архитектуре (MVC)**:

| Слой             | Пакет          | Назначение                                              |
| ---------------- | -------------- | ------------------------------------------------------- |
| **Domain**       | `model`        | Игровые сущности, бизнес-логика, генерация уровней      |
| **Presentation** | `presentation` | Рендеринг через Lanterna, экраны, пользовательский ввод |
| **Data**         | `datalayer`    | Сохранение и загрузка состояния игры через JSON (Gson)  |
| **Controller**   | `controller`   | Связывает слои: `MenuController`, `GameController`      |

Слой `domain` не зависит от `presentation` и `datalayer` — соблюдается принцип инверсии зависимостей.

## Что реализовано

### Доменный слой

- **Игровые сущности:** `GameSession`, `Level`, `Room`, `Corridor`, `Sector`, `Cell`, `Player`, `Backpack`
- **Характеристики персонажа:** здоровье, максимальное здоровье, ловкость, сила, текущее оружие
- **Рюкзак:** хранит до 9 единиц каждого типа предметов; сокровища накапливаются в одной ячейке

### Предметы

| Тип          | Эффект                                              |
| ------------ | --------------------------------------------------- |
| 🍖 Еда       | Восстанавливает здоровье                            |
| 🧪 Эликсир   | Временно повышает ловкость / силу / макс. здоровье  |
| 📜 Свиток    | Постоянно повышает ловкость / силу / макс. здоровье |
| ⚔️ Оружие    | Меняет формулу урона; при смене падает на пол рядом |
| 💎 Сокровище | Начисляется за победу над врагом, влияет на рейтинг |

### Противники

| Символ        | Тип        | Особенности                                                                |
| ------------- | ---------- | -------------------------------------------------------------------------- |
| `z` (зелёный) | Зомби      | Высокое HP, средняя сила, низкая ловкость                                  |
| `v` (красный) | Вампир     | Крадёт макс. здоровье; первый удар по нему — всегда промах                 |
| `g` (белый)   | Привидение | Телепортируется, периодически становится невидимым                         |
| `O` (жёлтый)  | Огр        | Ходит на 2 клетки; после атаки отдыхает, затем гарантированно контратакует |
| `s` (белый)   | Змей-маг   | Ходит по диагонали; может усыпить игрока на ход                            |

Каждый враг имеет собственный паттерн патрулирования. При обнаружении игрока — преследует кратчайшим путём.

### Боевая система

Бой пошаговый, инициируется движением в клетку врага. Каждый удар считается в три этапа:

1. **Проверка попадания** — вероятность зависит от ловкости атакующего и цели
2. **Расчёт урона** — зависит от силы и экипированного оружия
3. **Применение урона** — при HP ≤ 0 персонаж или враг погибает

### Генерация уровней

- Каждый уровень — сетка **3×3 секции**, в каждой случайная комната с произвольным размером и позицией
- Комнаты соединяются коридорами; граф проверяется на связность (`ConnectivityValidator`)
- На каждом уровне выделяются **стартовая** (без врагов) и **конечная** комнаты
- С глубиной растёт количество и сила врагов, снижается число полезных предметов

### Рендеринг и туман войны

- Отображение стен, пола, проёмов, коридоров, акторов и предметов
- **Туман войны:**
  - Неизведанные области — скрыты
  - Посещённые комнаты вне зоны присутствия — только контуры стен
  - Текущая комната — полная видимость
  - Из коридора — видимость через **Ray Casting** + алгоритм **Брезенхёма**
- **Экраны:** главное меню, игровой экран, инвентарь, таблица рекордов, экран завершения игры

### Сохранение

- Текущая сессия сохраняется в `saves/current_session.json` после каждого пройденного уровня
- Статистика всех попыток хранится в `saves/statistics.json`
- При загрузке восстанавливается **полное состояние**: позиции всех сущностей, характеристики персонажа, содержимое рюкзака

## Управление

| Клавиша         | Действие                                                      |
| --------------- | ------------------------------------------------------------- |
| `W` `A` `S` `D` | Движение (атака — движение в сторону врага)                   |
| `H`             | Выбрать и экипировать оружие из рюкзака (`0` — убрать оружие) |
| `J`             | Съесть еду из рюкзака                                         |
| `K`             | Выпить эликсир из рюкзака                                     |
| `E`             | Прочитать свиток из рюкзака                                   |

При использовании предмета выводится список доступных (1–9), игрок выбирает нужный.

## Запуск

**Требования:** JDK 21+, Apache Maven 3.6+

### Собрать и запустить через Maven

```bash
cd src/rogue21
mvn compile exec:java -Dexec.mainClass="reneakar.Main"
```

### Собрать JAR и запустить

```bash
cd src/rogue21
mvn package
java -jar target/rogue21-1.0-SNAPSHOT.jar
```

### Запустить тесты

```bash
cd src/rogue21
mvn test
```

## Структура проекта

```
src/rogue21/
├── pom.xml
├── saves/
│   ├── current_session.json     # Сохранение текущей сессии
│   └── statistics.json          # История всех прохождений
└── src/main/java/reneakar/
    ├── Main.java
    ├── controller/
    │   ├── MenuController.java
    │   └── GameController.java
    ├── model/                   # Доменный слой
    │   ├── GameSession.java
    │   ├── entities/            # Player, Enemy, Item, Room, Level, Corridor...
    │   │   ├── behaviors/       # AttackerCharacter, MoveableCharacter...
    │   │   ├── enemies/         # Zombie, Vampire, Ghost, Ogre, SnakeMage
    │   │   └── items/           # Food, Elixir, Scroll, Weapon, Treasure
    │   ├── enums/               # CellType, EnemyType, ItemType, GameState...
    │   ├── services/
    │   │   └── levelgenerator/  # Генератор уровней, ConnectivityValidator
    │   ├── utils/               # BresenhamLine, VisibilityCalculator
    │   └── valueobjects/        # Position, RoomConnection
    ├── presentation/            # Слой рендеринга (Lanterna)
    │   ├── Front.java
    │   └── screens/             # MainScreen, GameScreen, LeaderboardScreen...
    └── datalayer/               # Слой данных (Gson)
        ├── SaveManager.java
        ├── Statistics.java
        └── adapters/            # Gson-адаптеры для полиморфных типов
```
